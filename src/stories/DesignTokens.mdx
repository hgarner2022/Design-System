## Dark Mode

Our design system supports dark mode with specific color adjustments:

```css
/* Dark mode brand colors */
--color-brand: ${tokens.dark.color.brand.$value};
--color-brand-hover: ${tokens.dark.color['brand-hover'].$value};

/* Dark mode surface colors */
--color-surface: ${tokens.dark.color.surface.$value};
--color-surface-inverse: ${tokens.dark.color['surface-inverse'].$value};
--color-on-surface: ${tokens.dark.color['on-surface'].$value};
--color-on-surface-inverse: ${tokens.dark.color['on-surface-inverse'].$value};
```

## Usage

You can use these tokens in your CSS through our custom properties:

```css
.my-component {
  color: var(--color-primary-500);
  font-size: var(--font-size-md);
  padding: var(--space-4);
  border-radius: var(--radius-md);
  background-color: var(--color-surface);
}
```

Or in your components using our theme utilities:

```tsx
import { theme } from '@company/design-system'

function MyComponent() {
  return (
    <div
      style={{
        color: theme.colors.primary[500],
        fontSize: theme.fontSizes.md + 'px',
        padding: theme.spacing[4] + 'px',
        borderRadius: theme.radius.md + 'px',
      }}
    >
      Content
    </div>
  )
}
```

## Token Structure

Our tokens follow the [Design Tokens Format](https://design-tokens.github.io/) specification:

- **$type**: Defines the token type (color, dimension, fontFamily, etc.)
- **$value**: Contains the actual value
- **Nested structure**: Tokens are organized by category and subcategory
- **Theme support**: Separate light and dark theme tokens
