#### Surface Color Usage

Surface colors define the background and foreground colors for different contexts in our interface. They ensure proper contrast and accessibility across all components.

**Surface Colors:**
- **`surface`** - Primary background color for cards, panels, and main content areas
- **`surface-inverse`** - Dark background for inverted contexts (like dark mode or highlighted sections)
- **`on-surface`** - Text color that appears on light surfaces
- **`on-surface-inverse`** - Text color that appears on dark surfaces

**Usage Examples:**

```css
/* Card background */
.card {
  background-color: var(--surface);
  color: var(--on-surface);
}

/* Dark mode or highlighted section */
.highlighted {
  background-color: var(--surface-inverse);
  color: var(--on-surface-inverse);
}

/* Modal overlay */
.modal-backdrop {
  background-color: rgba(0, 0, 0, 0.5);
}

/* Tooltip */
.tooltip {
  background-color: var(--surface-inverse);
  color: var(--on-surface-inverse);
}
```

## Usage

You can use these tokens in your CSS through our custom properties:

```css
.my-component {
  color: var(--color-primary-500);
  font-size: var(--font-size-md);
  padding: var(--space-4);
  border-radius: var(--radius-md);
  background-color: var(--color-surface);
}

/* Semantic color usage */
.success-message {
  color: var(--color-success-500);
  background-color: var(--color-success-50);
}

.warning-alert {
  color: var(--color-warning-500);
  background-color: var(--color-warning-50);
}

.error-state {
  color: var(--color-error-500);
  background-color: var(--color-error-50);
}

.info-notice {
  color: var(--color-info-500);
  background-color: var(--color-info-50);
}
```

Or in your components using our theme utilities:

```tsx
import { theme } from '@company/design-system'

function MyComponent() {
  return (
    <div
      style={{
        color: theme.colors.primary[500],
        fontSize: theme.fontSizes.md + 'px',
        padding: theme.spacing[4] + 'px',
        borderRadius: theme.radius.md + 'px',
      }}
    >
      Content
    </div>
  )
}
```

## Token Structure

Our tokens follow the [Design Tokens Format](https://design-tokens.github.io/) specification:

- **$type**: Defines the token type (color, dimension, fontFamily, etc.)
- **$value**: Contains the actual value
- **Nested structure**: Tokens are organized by category and subcategory
- **Theme support**: Separate light and dark theme tokens
